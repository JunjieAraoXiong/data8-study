Choosing a chart

Every visualization question starts the same way: classify the variables, then read the pairing off a table. The rest is knowing the two or three properties of each chart that exams actually test.

Step 1

Classify each variable

Everything downstream depends on this call. The test is not "is it a number?" — it's "does arithmetic on it mean anything?"

Step 4

The recipes — question to chart, step by step

Seven workflows that cover nearly every visualization task in the course. The step people skip is almost always Prepare — most charts need a table operation first.

Count categories → bar chart

“How many cones of each flavor?”

  1. Classify. Flavor is categorical → bar chart.
  2. Prepare. The counts don't exist yet — build them:counts = cones.group('Flavor')
  3. Plot.counts.barh('Flavor')
  4. Read. Longest bar = most common. Order is arbitrary — never read a “trend” across bars.

A value per category → bar chart

“What's the average price per flavor?”

  1. Classify. Flavor categorical + Price numerical → one bar per flavor.
  2. Aggregate. select first, so the collect function only touches Price:avg = cones.select('Flavor', 'Price').group('Flavor', np.average)
  3. Plot. Mind the new label:avg.barh('Flavor')
  4. Read. Bar length = the average; the column is now called Price average.

One column's distribution → histogram

“How are the prices spread out?”

  1. Classify. Price is numerical → histogram.
  2. Choose bins. Equal widths unless you have a reason:bins = np.arange(3, 6.5, 0.5)
  3. Plot.cones.hist('Price', bins=bins, unit='dollar')
  4. Read. The y-axis is percent per dollar — compare areas, and remember they sum to 100%.

Distributions across groups → overlaid histograms

“Do smokers' babies weigh less?” — the book's ch. 12 example

  1. Classify. Birth Weight numerical + Maternal Smoker categorical.
  2. Plot — no prep needed:births.hist('Birth Weight', group='Maternal Smoker')
  3. Read. Two translucent histograms — look for a shift. Overlap doesn't mean “no difference”; deciding that is the A/B test's job.

Two numerical variables → scatter

“Do movies fans love also get critic love?” — sp25 midterm

  1. Classify. Critics and Fans are both numerical → scatter.
  2. Check scales. Different scales? Rescale first (sp25 made you do exactly this):imdb = imdb.with_columns('Fans Rescaled', imdb.column('Fans') / 10)
  3. Plot.imdb.scatter('Critics', 'Fans Rescaled', fit_line=True)
  4. Read. Direction, strength, shape. A curved cloud says don't trust the line — and association is never causation.

Change over time → line plot

“How many top movies came out each year?”

  1. Classify. Year is numerical and ordered → line plot.
  2. Prepare. One value per year:per_year = movies.group('Year')
  3. Plot.per_year.plot('Year', 'count')
  4. Read. Trends, peaks, turning points — claims about change, never about causes.

Answer “how many?” from a histogram

“The 61–63 inch bar has height 5; n = 300 students.” — su23 midterm

  1. Width. 63 − 61 = 2 inches.
  2. Percent. height × width = 5 × 2 = 10%.
  3. Count. percent × n = 0.10 × 300 = 30 students.
  4. Check. Only valid when the question lines up with bin edges — bins are [left, right).

the causal baitsu23 Midterm Q4

Does higher wind speed lead to more rentals? Scatter, histogram, line plot — or none of the above?

click to reveal
None of the above. “Lead to” is a causal claim; observational charts show association only. The scatter option is the trap — it shows the association, not the cause.

density arithmeticsu23 Midterm Q3

A histogram of 300 students' heights has its y-axis label missing. What is the label — and how many students are at least 61 but under 63 inches, if that bar's height is 5?

click to reveal
“Percent per inch” — never plain “percent.” Count = height × width × n = 5 × 2 × 300 ÷ 100 = 30 students. Same exam: merging two adjacent equal-width bars of heights x and y gives height (x+y)/2.
true / falsesp23 Final Q1a

“The height of each bar in a histogram represents the proportion of data within the corresponding bin.”

click to reveal
False. Height is density (percent per unit). The area — height × width — is the proportion. This exact sentence shows up as a T/F again and again.
reading a plotfa24 Final Q5c

A line plot shows counts of #1 songs per genre per year, with one line per genre. What kind of claim can a good title make?

click to reveal
A trend claim — the correct answer was “genres of #1 songs have diversified over time.” Titles asserting causes or predictions overreach what a line plot shows.
lookalike trapsp25 Midterm Q1i

From lecture's du_bois table: percent of income spent on food, for income classes 100–200, 200–300, … “1000 and over.” Histogram, overlaid histogram, overlaid bar chart — or bar chart?

click to reveal
Bar chart. The labels look numerical, but “Income Class” is a set of category labels — the bars are separated, and the y-axis is a value per category, not a density. One series, so nothing is “overlaid.” The whole question is the Step-1 classification call.
classify variablessp25 Midterm Q2a

An imdb table has ID (integer), Title (string), Year (integer), Genre (string), Critics (float), Fans (integer), Runtime (integer). Which attributes are numerical — select all that apply?

click to reveal
Year, Critics, Fans, Runtime. ID is stored as an integer, but it's a code that identifies a movie's URL — averaging IDs means nothing, so it's categorical. The “integer ⇒ numerical” reflex is exactly what this question punishes (same trap as zip codes).
density arithmeticsp25 Midterm Q2c

A histogram of movie runtimes uses 10-minute bins (…130–140, 140–150, 150–160, 160–170…), y-axis “percent per minute.” Which can you answer from it: (a) % of movies ≥ 140 min · (b) % > 160 min · (c) % < 160 min · (d) % at least 125 but under 130?

click to reveal
(a) and (c) only. Questions must line up with bin edges: bins are [left, right), so “≥ 140” and “< 160” are sums of whole bars. “> 160” silently drops the exactly-160 movies living inside the 160–170 bin, and 125–130 would split a bin down the middle.
